# PRD：AI 营销文案生成功能（MVP 核心模块）

---

## 一、需求背景

本地中小门店商家普遍面临「不会写文案、没时间写」的痛点，导致私域运营停滞、客户流失。AI 文案生成功能是产品付费转化的核心驱动模块，直接决定商家的留存和复购意愿。

---

## 二、目标用户

| 用户类型 | 说明 |
|---|---|
| 免费版用户 | 每日限 5 次生成，体验核心流程，引导付费转化 |
| 付费版用户（基础/高级会员） | 无限次生成，解锁全部行业模板和平台适配 |

---

## 三、核心用户故事

1. 作为**餐饮店老板**，我希望**选完行业和场景后只需填门店名和活动信息就能出文案**，以便**不用自己想，直接复制发朋友圈**。
2. 作为**美甲店主**，我希望**生成适配小红书风格的文案并带上同城话题标签**，以便**发出去能吸引同城客户进店**。
3. 作为**教培机构运营**，我希望**把满意的文案收藏起来下次直接用**，以便**不用每次重新生成，节省时间**。
4. 作为**免费版用户**，我希望**用完 5 次后看到清晰的升级引导**，以便**清楚知道付费能解锁什么权益**。
5. 作为**付费用户**，我希望**对同一场景一键重新生成多个版本**，以便**挑选最满意的一条发布**。

---

## 四、功能详细说明

### 模块 1：行业与场景选择

**功能名称**：行业/场景级联选择器

**交互逻辑**：
1. 页面顶部展示行业选项卡：餐饮 / 美业 / 教培（后续可扩展）
2. 选中行业后，下方以卡片网格形式展示该行业下 8-10 个场景（图标 + 场景名称）
3. 点击场景卡片，右侧/下方展开填空表单区域

**边界条件**：
- 免费版用户当日已用满 5 次时，点击任意场景卡片弹出升级弹窗，不进入填空步骤
- 默认选中「餐饮」行业，不允许不选行业直接进入下一步

**前端展示**：
- PC：左侧固定行业 Tab + 右侧场景卡片网格（3 列）
- H5：行业横向滚动 Tab + 场景卡片 2 列网格
- 组件：`<IndustryTabs>`、`<SceneGrid>`

---

### 模块 2：商家信息填空表单

**功能名称**：场景参数填写表单

**交互逻辑**：
1. 根据所选场景动态渲染填空项，每个场景固定字段：门店名称、活动内容、优惠力度、活动时间、门店地址（部分场景可选）
2. 提供两个全局设置项：
   - **文案风格**：接地气口语 / 高级氛围感 / 促销紧迫感 / 干货科普（单选 Radio）
   - **适配平台**：朋友圈 / 小红书 / 抖音 / 社群（单选 Radio，影响文案格式和标签风格）
3. 点击「生成文案」按钮提交，进入生成等待状态

**边界条件**：
- 门店名称为必填项，其余字段为选填（未填则 Prompt 中跳过该变量）
- 门店名称最长 20 字，活动内容最长 100 字，超出截断并提示
- 「生成文案」按钮在生成进行中禁用，防止重复提交

**前端展示**：
- 使用 Ant Design `Form` 组件 + `Input` / `Radio.Group`
- H5 端输入框 `font-size` ≥ 16px，防止 iOS 自动缩放
- 组件：`<SceneForm>`

---

### 模块 3：文案生成与展示

**功能名称**：AI 文案生成与结果展示

**交互逻辑**：
1. 点击「生成文案」后，结果区域出现打字机动效（SSE 流式输出）
2. 生成完成后展示完整文案，并显示操作按钮区：一键复制 / 重新生成 / 收藏
3. 点击「一键复制」将文案写入剪贴板，按钮文字变为「已复制 ✓」，2 秒后恢复
4. 点击「重新生成」保持当前填空内容不变，重新调用接口，结果区域内容刷新（不计入免费次数，仅首次生成计数）
5. 生成失败时展示错误提示，提供「重试」按钮

**边界条件**：
- 单次生成超时时间 15s，超时后提示「生成超时，请重试」
- 免费版用户每次点击「生成文案」（非重新生成）消耗 1 次配额；配额清零时间为每日 00:00
- 重新生成不限次数（免费版和付费版均不限）

**前端展示**：
- 结果区域 `<CopywriterResult>`：文本展示 + 操作按钮栏
- 加载态：骨架屏 + 打字机光标动效
- 组件：`<CopywriterResult>`、`<ActionBar>`

---

### 模块 4：收藏与历史记录

**功能名称**：文案收藏 & 历史列表

**交互逻辑**：
1. 点击「收藏」将当前文案保存至「我的收藏」
2. 「历史记录」入口在个人中心，列表按生成时间倒序展示，每条记录显示行业、场景、文案摘要（前 50 字）
3. 点击历史记录条目可展开查看完整文案，并支持再次复制/收藏

**边界条件**：
- 免费版最多保存 20 条历史记录，超出后自动删除最旧的一条；付费版不限
- 收藏上限：免费版 10 条，付费版不限
- 删除收藏/历史需二次确认弹窗

**前端展示**：
- 组件：`<HistoryList>`、`<CopywriterCard>`

---

### 模块 5：免费次数引导升级

**功能名称**：配额限制与付费引导

**交互逻辑**：
1. 页面顶部常驻显示今日剩余次数（如「今日剩余：3 次」）
2. 剩余 1 次时，生成结果下方出现软性引导横幅「升级会员，无限生成」
3. 次数用尽后，「生成文案」按钮变为「升级解锁」，点击直达会员购买页

**边界条件**：
- 付费版用户不展示次数限制相关 UI
- 升级弹窗需展示免费版 vs 付费版权益对比

**前端展示**：
- 组件：`<QuotaBanner>`、`<UpgradeModal>`

## 五、验收标准（AC）

**AC1：正常生成文案**
```gherkin
Given 用户已选择「餐饮」行业「朋友圈日常种草」场景
  And 已填写门店名称「老王麻辣烫」和活动内容「新品上市」
  And 选择「接地气口语」风格和「朋友圈」平台
When 点击「生成文案」按钮
Then 结果区域以打字机动效逐字展示文案
  And 文案包含门店名称和活动内容
  And 生成完成后「一键复制」「重新生成」「收藏」按钮可点击
```

**AC2：免费版次数限制**
```gherkin
Given 免费版用户今日已生成 5 次文案
When 再次点击「生成文案」
Then 弹出升级引导弹窗
  And 弹窗展示免费版与付费版权益对比
  And 不触发 AI 接口调用，不消耗 API token
```

**AC3：重新生成不消耗配额**
```gherkin
Given 免费版用户今日已使用 4 次，当前页面有生成结果
When 点击「重新生成」
Then 结果区域内容刷新为新文案
  And 今日剩余次数仍显示 1 次（未被消耗）
```

**AC4：一键复制**
```gherkin
Given 页面展示了生成完成的文案
When 点击「一键复制」
Then 文案内容写入用户剪贴板
  And 按钮文字变为「已复制 ✓」
  And 2 秒后按钮恢复为「一键复制」
```

**AC5：生成超时处理**
```gherkin
Given 用户点击「生成文案」
When AI 接口 15 秒内未返回结果
Then 结果区域展示「生成超时，请重试」提示
  And 出现「重试」按钮
  And 本次操作不消耗免费配额
```

## 七、依赖与风险

**外部依赖：**
- DeepSeek API / 字节豆包 API：文案生成核心依赖，需提前申请 API Key 并充值
- 后端限流服务（Redis）：免费版每日配额计数依赖 Redis，需提前部署

**风险点：**

| 风险 | 影响 | 应对方案 |
|---|---|---|
| 大模型 API 响应慢（>5s） | 用户体验差，流失率上升 | 使用流式 SSE 输出，首字符 ≤1s 即可保留用户 |
| 大模型 API 服务中断 | 核心功能不可用 | 接入 2 个大模型作为主备（DeepSeek 主，豆包备） |
| Prompt 输出质量不稳定 | 商家对文案不满意，影响留存 | MVP 前需人工验收每个场景至少 20 次生成结果 |
| iOS Safari 剪贴板权限限制 | 一键复制在 H5 失效 | 降级方案：复制失败时展示文案选中框，引导手动复制 |
